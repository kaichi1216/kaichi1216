ruby 和 rails中的 Symbol

2019-09-19

前後端開發 css Ruby 0n Rails 菜鳥日誌

剛開始學ruby的時候，看到一個變數前面有個 :符號，那時想說這到底是什麼鬼東西，在寫的時候也很疑惑， 他跟變數有什麼差別，跟什麼時機要使用它呢？
上圖的意思是happy是一個變數 然後他指向給寫程式一時爽，一直寫一直爽，變數跟Symbol不一樣的地方是，沒有寫程式一時爽，一直寫一直爽這個字串指向 happy的話，happy是沒辦法單獨存在的，但Symbol可以，因為他本身不需指向任何東西也可以拿來使用

Symbol 跟字串的差別？

Symbol跟字串蠻相似的 ，不過Symbol是不可變的字串(immutable)意思就是不能被修改的，還有就是Symbol的效能比字串稍微好一點 如下圖 字串的話，每次都會產生不同的記憶體，等於是五個不同的物件存在記憶體了，Symbol就都是相同的物件id，相對的就節省記憶體的空間 要怎麼知道哪時要使用字串或symbol呢？ 比較常見的是 Hash裡面的Key，因為Symbol的不可變的特性，查詢速度快，字串的使用時機例如 想把畫面上的內容印出來就可以使用，因為如果使用Symbol的話，還是要先轉成字串在印出。
