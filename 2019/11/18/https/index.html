HTTP 與 HTTPS

2019-11-18

Ruby 0n Rails 菜鳥日誌

在網址連結上HTTP與HTTPS的差別不單只是1個英文字S的差異，這僅一個字母S的差別卻代表了網站使用編碼協定的安全性(secure)，http://跟https://之間的不同在於網路文字傳送協定中標準的不同，在網址欄位上連結為https開始的網址會多顯示”安全”標示，讓您簡單的瞭解網站對於使用者的友善性。

http的網站在傳輸資料時，不像https會做資料的加密，等於是把資料公開的直接傳向目的地，中間的站點就有機會被攔截或被看到甚至記錄下來，舉個例子，你傳紙條給喜歡的女生小穎，假如你坐最後一排，她坐第一排，中間經過小王 他也喜歡小穎，這時使用http因為是公開的，他就可以在裡面加料，傳到小穎手上時，這時後果不是一巴掌就是一輩子(疑！！)。

但如果你將紙條裝在一個有鎖的盒子中，而鑰匙只有你喜歡的女生有，那中間的同學就無法輕易看到你們的傳的訊息。https就是這樣，他會將訊息加密後再傳送出，而解碼的方式只有目標站台才有，因此來得安全很多。

那麼什麼是HTTPS & SSL

  • HTTPS (超文字安全傳輸通訊協定) 是一種網際網路通訊協定，可確保資料在使用者的電腦和網站之間傳輸時，保有完整性和機密性，提供三道重要的資安防護網
  • SSL(Secure Sockets Layer)用於保持網際網路連線安全以及防止在兩個系統之間發送的所有敏感資料被罪犯讀取及修改，包括潛在的個人詳細資料。兩個系統間 (例如購物網站與瀏覽器)，或者伺服器至伺服器 (例如，建立會員、購物車、金流、刷卡機制服務的網站)。
